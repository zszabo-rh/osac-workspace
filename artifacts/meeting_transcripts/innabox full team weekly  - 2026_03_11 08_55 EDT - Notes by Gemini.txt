Summary
Slurm proposal comments require review, bare metal architecture issues discussion postponed, and a decision made to use automation for integrating Jira and GitHub issue tracking.

Slurm Proposal and Architecture
The Slurm enhancement proposal was introduced with a plan for Lars Kellogg-Stedman and Orran Krieger to review and respond to final comments regarding the proposal. The bare metal architecture issues discussion was postponed until Michael can attend the meeting.

Issue Management Tooling Integration
The group decided to use automation to create and connect GitHub issues to Jira to merge internal Red Hat tools with public-facing tools, ensuring all work is public. Crucial issues for the rollout of cluster as a service and bare metal as a service will be made visible as GitHub issues.

Pluggable Inventories and Integration
Progress on pluggable inventories is urgently needed for integration with Carbide and Netris, though the API cannot be finalized until bare metal work is complete. The group agreed on the value of creating a common API before integrating Carbide or Netris to inform the design.
